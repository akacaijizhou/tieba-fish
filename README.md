# Tieba Fish

一个面向 VS Code 使用场景的贴吧阅读插件。

Tieba Fish 的定位不是完整贴吧客户端，而是把“看帖”这件事收成更适合编辑器环境的一层阅读体验。  
它保留常用入口、正文阅读、翻页和临时隐藏这些高频动作，尽量减少网页式浏览的干扰。

## 产品定位

这个插件主要处理 3 类需求：

- 在工作流里顺手看帖，不频繁切浏览器
- 正文优先，减少原站界面噪音
- 在被打断时快速切换和恢复

当前产品结构也围绕这 3 点展开：

- 左侧提供 `关注吧 / 最近 / 收藏 / 历史`
- 右侧用阅读器方式展示帖子正文
- 状态栏和命令面板承接常用动作

## 适用场景

- 日常把 VS Code 作为主界面，希望减少窗口切换
- 更关注“看内容”和“继续阅读”，而不是完整贴吧交互
- 经常看长帖，希望只看楼主、跳页、看楼中楼更顺手
- 需要在办公室场景下快速切换视图

## 核心能力

### 内容入口

- `关注吧`：手动添加或同步自己常看的吧
- `最近`：显示最近一次打开的关注吧列表
- `收藏`：保存想回看的帖子
- `历史`：保存最近打开过的帖子

### 阅读体验

- 以正文为主的帖子排版
- `只看楼主`
- 楼中楼预览、展开和分页
- 图片缩略图和大图预览
- 底部跳页
- 回到顶部

### 使用效率

- `阅读: 继续阅读`
- `阅读: 浏览指定链接`
- 帖子页内快捷键帮助
- 状态栏主题预设切换
- `Ctrl+Alt+X` 老板模式

## 安装方式

### 扩展市场

在 VS Code 扩展市场搜索：

- `Tieba Fish`

### VSIX 安装

如果使用离线安装包，可以在 VS Code 中：

1. 打开扩展视图
2. 点击右上角 `...`
3. 选择 `Install from VSIX...`
4. 选择对应的 `.vsix` 文件

## 建议的首次使用路径

1. 执行 `开始使用: 打开首页`
2. 先执行 `开始使用: 添加贴吧`，或者 `阅读: 浏览指定链接`
3. 打开一个吧，再打开一个帖子开始阅读
4. 如果想同步账号里的关注吧，再执行 `开始使用: 导入登录态`
5. 如果想补更稳的结构化阅读链路，再执行 `开始使用: 安装 aiotieba`

这个顺序的目的，是先快速进入阅读，再按需要补登录态和环境能力。

## 登录态说明

当前最稳的方式是直接导入完整 Cookie。原因很简单：

- 阅读链路更稳定
- 同步关注吧需要完整登录态
- 插件会自动提取需要的字段
- 回退到网页方式时也更容易复用

### 获取 Cookie 的方式

1. 在浏览器里打开贴吧，并确认自己已经登录
2. 按 `F12` 打开开发者工具
3. 进入 `网络 / Network`
4. 选择任意一个贴吧请求
5. 在请求头中找到 `Cookie`
6. 复制完整 Cookie
7. 回到 VS Code，执行 `开始使用: 导入登录态`

登录态会保存在 VS Code Secret Storage，不会写入普通设置。

## 左侧四个视图分别做什么

### 关注吧

用于维护长期入口：

- 手动添加贴吧
- 同步账号已关注的贴吧
- 进入某个吧的帖子列表

### 最近

`最近` 不是全站推荐流。  
它表示最近一次打开的关注吧列表，适合沿着当前阅读上下文继续往下看。

### 收藏

用于沉淀想回看的内容：

- 长帖
- 资料贴
- 稍后再看的帖子

在帖子阅读页里点一次 `收藏`，这里就会按时间倒序保存。

### 历史

用于回到最近看过的帖子。  
如果中途关闭了正文，再回来时通常可以直接使用：

- `阅读: 继续阅读`

如果上次停在 `只看楼主`，继续阅读也会一起恢复这个状态。

## 帖子页里的常用动作

打开帖子后，常用动作集中在阅读本身：

- `收藏 / 取消收藏`
- `只看楼主`
- 展开楼中楼回复
- 楼中楼翻页
- 切换图片显示
- 点击图片查看大图
- 直接跳页
- 回到顶部
- `?` 打开快捷键帮助

如果当前走的是网页回退链路，楼中楼会保留预览，但会明确标成 `仅预览`，需要完整展开时可以直接用：

- `VS Code 浏览器`
- 系统浏览器

## 快捷键帮助

打开方式：

- 执行 `阅读: 快捷键帮助`
- 在帖子页直接按 `shift + ?`

当前主要快捷键：

- `R` 刷新当前帖子
- `L` 切换只看楼主
- `I` 显示或隐藏图片
- `J` 下一页
- `K` 上一页
- `G` 聚焦跳页输入框
- `Esc` 关闭帮助或图片预览
- `Ctrl+Alt+X` 老板键

## 主题预设

状态栏会显示当前主题，例如：

- `主题: 默认`

点击后可以在 3 个预设之间切换：

- `默认`
- `极简`
- `文档风`

它们分别对应常规阅读、低存在感界面和长文阅读场景。

另外在扩展设置里还可以直接控制：

- `tieba.compactMode`：进一步压缩阅读密度
- `tieba.lowContrastMode`：弱化分隔线、按钮和反馈条的视觉存在感

## 老板模式

快捷键：

```text
Ctrl+Alt+X
```

触发后会：

- 把 Tieba 侧边栏切成伪造文件树
- 把正文页切成伪造代码文件

再次触发会恢复到原来的贴吧内容。

## 常见问题

### 能看帖子，但同步关注吧失败

通常说明当前登录态不完整。  
建议重新执行：

- `开始使用: 导入登录态`

然后直接粘贴完整贴吧 Cookie。

### 为什么有时会回退到浏览器

贴吧本身存在安全校验。  
在少数情况下，插件会回退到网页方式继续打开内容，这是兜底行为。

在网页回退链路下，帖子正文仍然可以看，但楼中楼通常只保留预览，不保证能完整展开。

### 没有 Python 能不能用

可以直接开始用。  
当前会优先尝试 `aiotieba`，不满足时再回退到网页链路继续阅读。

如果本机没有 Python，可以先执行：

- `开始使用: 下载 Python`

但这不是第一次使用的前置条件。

### 首页和环境诊断分别做什么

- 首页负责告诉你当前状态和下一步动作
- 环境诊断负责说明当前卡在哪一层

如果只是想尽快用起来，先看首页。  
如果已经装过、配过，但仍然有问题，再看环境诊断。

### 一键安装 aiotieba 失败怎么办

也可以手动执行：

```powershell
python -m pip install aiotieba
```

安装后重开 VS Code，或者重新打开 `维护: 环境诊断`。

## 已知边界

- 贴吧本身存在安全校验，个别页面可能仍会失败或回退
- `最近` 不是推荐流，而是最近一次打开的关注吧数据
- 网页回退链路下，楼中楼通常只展示预览
- 部分能力依赖完整登录态和本机 Python 环境
