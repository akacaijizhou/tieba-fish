# Tieba Fish 产品路线图

## 1. 当前产品判断

Tieba Fish 不是贴吧客户端，而是 VS Code 里的低打扰贴吧阅读器。

当前阶段不继续扩功能面，而是只做一轮交互和认知层优化，让用户：

- 第一次打开就知道下一步该点什么
- 日常使用时能快速理解当前状态
- 阅读时加载、筛选、切换都有明确反馈
- 在不增加复杂度的前提下，获得更强的降噪和伪装能力

这轮只保留 8 个方向：

- 关键词屏蔽
- 快捷键帮助
- 主题预设
- 空态文案更强引导
- 首页入口再收敛
- 状态提示再人话一点
- 加载反馈统一
- 命令入口分层

## 2. 本轮不做

本轮明确不做这些方向：

- 阅读位置恢复到楼层
- 楼层导航继续增强
- 稍后看
- 吧内搜索
- 连续看帖切换
- 发帖、回复、点赞、社交互动

原因很直接：这些都不是当前用户最痛的断点。当前最需要解决的是“能不能更顺手、更低打扰、更容易理解”。

## 3. 产品目标

本轮结束后，产品应达到这 4 个目标：

1. 新用户第一次进入时，不看文档也知道先做什么。
2. 老用户看到状态栏、空态页、加载态时，不需要理解技术字段也能知道当前是否可用。
3. 常见操作过程中没有“像卡住了一样”的无反馈状态。
4. 在阅读体验上增加最小但有效的降噪能力，而不是继续堆功能。

## 4. 优先级

### P0：先解决理解成本和操作反馈

- 首页入口再收敛
- 状态提示再人话一点
- 加载反馈统一
- 命令入口分层
- 空态文案更强引导

### P1：再补低打扰阅读能力

- 关键词屏蔽
- 快捷键帮助
- 主题预设

## 5. 版本建议

### V0.5：入口与认知优化

目标：让用户第一次打开和日常使用都更容易理解。

包含：

- 首页入口再收敛
- 状态提示再人话一点
- 加载反馈统一
- 命令入口分层
- 空态文案更强引导

### V0.6：低打扰能力补齐

目标：在不增加复杂度的前提下，提升阅读控制力。

包含：

- 关键词屏蔽
- 快捷键帮助
- 主题预设

## 6. 功能判断

### 6.1 首页入口再收敛

首页只保留最关键的动作，不再像功能列表。

建议保留：

- 导入贴吧登录态
- 同步我关注的贴吧
- 继续阅读或打开最近阅读
- 打开环境诊断

不建议把所有命令都堆到首页。

### 6.2 状态提示再人话一点

用户不需要理解 `BDUSS`、`STOKEN`、`aiotieba` 这些技术词才能判断当前状态。

建议统一成面向结果的表达：

- 可阅读
- 可同步关注吧
- 需要补全登录态
- 需要安装 Python
- 需要安装 aiotieba

技术细节只放到诊断页。

### 6.3 加载反馈统一

所有核心操作都要有一致的反馈样式。

重点覆盖：

- 关注吧加载
- 最新加载、翻页、刷新
- 历史加载
- 帖子翻页
- 只看楼主切换
- 楼中楼展开与翻页

### 6.4 命令入口分层

命令面板和文档里的命令不再按“技术实现”组织，而按用户心智组织：

- 开始使用
- 阅读
- 维护

目标是降低命令发现成本，而不是增加命令数量。

### 6.5 空态文案更强引导

空态不能只说“当前没有数据”，而要明确告诉用户下一步该点什么。

重点视图：

- 关注吧
- 最新
- 历史
- 首页引导

### 6.6 关键词屏蔽

这是最值得做的降噪功能之一。

建议第一版支持：

- 标题关键词屏蔽
- 作者名屏蔽
- 吧名屏蔽

第一版只做本地过滤，不做云同步。

### 6.7 快捷键帮助

现在已经有老板模式，但其他能力的可发现性很弱。

建议做一个轻量帮助面板，集中展示：

- 老板模式
- 帖子翻页
- 回到顶部
- 只看楼主
- 打开指定链接

### 6.8 主题预设

不做复杂主题配置，而是只做少量预设。

建议第一版只给 2 到 3 个：

- 默认
- 极简
- 文档风

目标是降低视觉存在感，而不是做个性化换肤系统。

## 7. 验收标准

本轮完成后，至少满足：

- 新用户第一次打开首页时，能在 30 秒内完成第一步操作判断
- 所有核心加载链路都能看到统一反馈
- 空态文案都包含明确下一步动作
- 命令面板的命令结构不再显得杂乱
- 关键词屏蔽能稳定影响列表与正文展示
- 快捷键帮助和主题预设都能被普通用户发现并使用

## 8. 推荐执行顺序

1. 首页入口再收敛
2. 状态提示再人话一点
3. 加载反馈统一
4. 命令入口分层
5. 空态文案更强引导
6. 关键词屏蔽
7. 快捷键帮助
8. 主题预设
